# 작업 지시서: ALFA 런북 어시스턴트 제작

> 이전 세션에서 완료된 작업 기반으로, 고객 미팅 시 실시간 어시스트하는 런북 형태 도구를 만든다.

---

## 목적

ALFA 프레임워크를 고객 미팅 현장에서 사용할 수 있는 **어시스턴트**로 만든다. 미팅 전/중/후에 활용.

## 만들어야 할 것

### 1. 미팅 전 — 질문 리스트 생성기

**입력:** 고객 프로필 (있는 만큼)
**출력:** 미팅에서 물어볼 질문 리스트 (고객 프로필 빈칸 채우기 위한)

**패턴 (시뮬레이션에서 추출):**
- 업종별로 중요한 질문이 다름 (금융→규제, 제조→온프레미스, 이커머스→고객대면)
- 고객 프로필 6항목 중 빈칸 기반으로 질문 생성
- 거버넌스 무게 예측 → 컴플라이언스 동석 필요 여부 판단

### 2. 미팅 중 — 실시간 프로필 업데이트

**입력:** 미팅 중 나온 내용 (메모 또는 음성 요약)
**출력:** 고객 프로필 업데이트 + 아직 안 물어본 것 알림

### 3. 미팅 후 — 산출물 자동 생성

**입력:** 업데이트된 고객 프로필
**출력:**
- 미팅 서머리 (고객용)
- 내부 정리 (ALFA Phase 진행 상태 + 다음 작업)
- ALFA 입력 문서 (프로필 확정 시)
- 유즈케이스 정의서 초안 (유즈케이스 식별된 경우)

### 4. IRM 투입 판단

**입력:** 확정된 유즈케이스 정의서
**출력:** IRM Phase 0 시작 가능 여부 + 예상 결과 (행위 유형 분포, GraphRAG 필요성)

## 미팅 대화 패턴 (시뮬레이션에서 추출)

```
공통 흐름:
1. 현황 듣기 (뭐가 문제야?)
2. 데이터/인프라 확인 (뭐가 있어?)
3. 사용자 확인 (누가 쓸 거야?)
4. 고객 대면 여부 확인 (내부? 외부?)
5. 스폰서/예산/일정 확인
6. 규제 확인 (업종별 다름)
7. 방향 제시 (거버넌스 먼저, 내부부터)
8. 로드맵 제시
9. 다음 스텝 합의

업종별 분기:
- 금융: 6번에서 "컴플라이언스팀 필요" → 2차 미팅
- IT서비스: 6번 스킵 가능 → 1회로 끝
- 제조: 2번에서 "온프레미스" → 로컬 LLM 방향
- 이커머스: 4번에서 "고객 대면" → 내부 먼저 + 2차 전환 경고
```

## 주의 사항

- ALFA 용어를 고객한테 쓰지 않음 (내부 도구)
- 미팅 보기(선택지)는 런북에 포함하되, 상황별로 자동 추천하는 수준으로
- IRM 용어도 고객한테 안 씀 (행위 유형, GraphRAG 같은 거)
- 고객 대면 여부가 거버넌스 무게를 결정하는 핵심 트리거 — 반드시 초반에 확인

---

## 검증 기준

- 시뮬레이션 4개 케이스를 런북으로 재현할 수 있는가
- 새로운 업종(병원, 공공)을 런북만으로 미팅 진행할 수 있는가
- 미팅 후 산출물이 ALFA 입력 문서 양식과 일치하는가
