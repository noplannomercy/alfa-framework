# 병원그룹 ALFA 입력 프로필 (확정)

> 확정일: 2026-03-25
> 버전: v1.0
> 미팅: 1~3차 완료

## 1. 회사 개요
- 업종: 의료 (병원 그룹)
- 규모: 중견 — 서울 본원 + 지방 분원 3개
- 주력 사업: 종합병원 운영
- 의사결정 기구: 의료정보위원회 (AI 관련 안건 심의)
- RACI: 실행=의료정보팀, 검토=영상의학과, 승인=의료정보위원회, 보고=CTO

## 2. AI/LLM 현황
- 기존 시도: 작년 외부 업체 챗봇 PoC — 중단
- 실패 원인: 범용 LLM의 의학용어 이해 부족 → 정확도 미달
- CTO 방향: 자체 데이터 기반, 외부 업체 의존 축소
- GPU: A100 x2 (연구팀 소유, 주중 업무시간 공유 합의 완료)

## 3. 인프라
- 운영 환경: 온프레미스 확정 (데이터 외부 반출 불가, 외부 API 불가)
- EMR: 자체개발, 온프레미스 서버
  - 분원 3곳 EMR 버전 상이, 데이터 구조 차이 있음
  - 접근권한: 기존 EMR 권한체계 연동
  - 감사로그: EMR 기존 감사로그에 AI 조회 이력 추가
- PACS: Infinitt 제품, HL7 FHIR 연동 가능, 읽기전용만 허용
- GPU: NVIDIA A100 x2 (주중 업무시간 우선 사용, 야간/주말 연구팀)
- 스토리지: 원내 서버 + 학습용 데이터 별도 격리 스토리지

## 4. 규제 환경
- 감독기관: 식약처, 보건복지부, 개인정보보호위원회
- 규제 담당: 의료정보보안팀장
- 데이터: 개인건강정보(PHI) — 가명처리 후 LLM 투입 가능 (내부 검토 완료)
- 가명처리: 보건복지부 가이드라인 — 이름/생년월일/주민번호 마스킹
- IRB: 간편심의 확정, 후향적 데이터, 2주 통과 예상
- 식약처: 보조수단 인정 ("AI 결과를 의사가 최종 확인하는 구조")
- 환자 대면: 현재 없음 (향후 2단계에서 검토)
- 외부 API: 절대 불가

## 5. 원하는 것
- 경영진: CTO "우리 데이터로 직접 해보자"
- 1순위 (PoC 확정): 본원 영상의학과 판독문 검색 보조
- 2순위: 간호기록 요약 (교대 인수인계, 현재 수기 30분)
- 핵심 Pain Point: 의사별 표현 차이로 판독문 검색 불가 (CT/컴퓨터단층촬영/영문약어 혼재)
- 장기 비전: 환자 설명용 활용

## 6. 제약사항
- 예산: PoC 5천만원 이내
- 일정: 4월 중순 시작 (IRB 2주 + 데이터 준비 2주)
- 인력: 의료정보팀 3명
- 범위: 본원만 (분원은 1차 제외)
- 사용자: 의사/간호사 전용
- 의사결정: CTO 스폰서, 의료정보위원회 승인
