## 기술 신호

- **거버넌스 무게**: Heavy (관리 가능)
  - 근거: 의료업종 + PHI + IRB, 단 보조수단 인정 + 간편심의 + 환자 비노출
  - 블로커: **없음** — 모든 규제 경로 확보됨
- **GraphRAG 필요**: **예 (강)**
  - 신호: "같은 소견인데 표현 다름", "CT/컴퓨터단층촬영/영문약어 혼재"
  - 이전 PoC 실패 원인과 동일 → 이번 PoC 핵심 차별화 요소
- **인프라 경로**: 온프레미스 로컬 LLM
  - A100 x2 (주중 업무시간), Infinitt PACS (HL7 FHIR 읽기전용)
  - 데이터: 후향적 판독문 → 가명처리 → 격리 스토리지
- **PoC 이력**: 있음 — 작년 외부 업체 범용 LLM 챗봇, 의학용어 미이해로 중단
  - CTO 기조 변화: "외부 의존 줄이고 우리 데이터로 직접"

## 감지된 리스크

| 리스크 | 심각도 | 대응 |
|--------|--------|------|
| A100 가용성 충돌 (연구팀 vs PoC) | 중 | 주중 업무시간 합의 완료, 야간/주말 제외. 본격 운영 시 GPU 증설 필요 |
| 분원 EMR 버전 차이 | 중 | 1차 PoC 본원 한정으로 회피. 분원 확대 시 데이터 정규화 레이어 필수 |
| 이전 PoC 실패 트라우마 | 중 | 차별점(자체 데이터 + 용어 매핑) 명확히 어필, 조기 성과 시연 필요 |
| 환자 대면 확장 시 거버넌스 격상 | 낮 (장기) | 2단계 진입 시 식약처 SaMD 경로 재검토, 별도 거버넌스 세션 필요 |

## 예상 로드맵

### 온프레미스 규제 Heavy (총 3~4개월)

```
[사전 준비] 3월 말 ~ 4월 중순 (4주)
  Week 1-2: IRB 간편심의 접수 및 통과
  Week 3-4: 가명처리 파이프라인 구축 + 격리 스토리지 세팅 + PACS 연동 테스트

[Phase 1 PoC] 4월 중순 ~ 6월 (8~10주)
  Week 1-2: A100 환경 세팅 + 로컬 LLM 배포 + 벡터DB 구축
  Week 3-4: 판독문 임베딩 + 용어 온톨로지/동의어 매핑 (GraphRAG 핵심)
  Week 5-6: RAG 파이프라인 + 판독문 검색 프로토타입
  Week 7-8: 영상의학과 전문의 피드백 + 정확도 개선
  Week 9-10: 2주 파일럿 운영 + 평가

[Phase 1.5] 6월 말 (+2주)
  간호기록 요약 데모 (같은 NLP 파이프라인 활용)

[Phase 2] 별도 예산/일정
  분원 확대 — EMR 데이터 정규화 + ETL 파이프라인

[Phase 3] 장기
  환자 대면 — 거버넌스 재검토 (Very Heavy)
```
