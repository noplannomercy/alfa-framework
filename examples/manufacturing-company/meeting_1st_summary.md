# 1차 미팅 서머리

> 일시: 2026-03-25
> 장소: 제조기업 (자동차 부품)
> 참석: 최차장 (품질관리팀), 아키텍트
> 목적: LLM 도입 검토 — 품질 불량 원인 분석 효율화

---

## 논의 내용

### 1. 현재 상황
- 불량 발생 시 원인 분석에 시간 과다 소요 (간단 1~2시간, 복잡 하루+, 최악 3일)
- 문서가 4곳에 분산 (품질 포털 PDF, 품질 DB, 공유폴더, 설비팀 서버)
- 경력 20년 박부장님에게 의존 (속인화 문제)
- 작년 고객 클레임 사건이 트리거 → 공장장 지시

### 2. 데이터/인프라
- 검사 기준서 100개 (PDF), 공정 매뉴얼 50개 (Word)
- 불량 이력 만 건+ (품질 DB, PostgreSQL 기반, MES 연동)
- 불량 이력에 증상/원인/조치 텍스트 있음 (표현 비통일 이슈)
- 온프레미스 전용 (데이터 외부 반출 금지)
- GPU 서버 A100 보유 (유휴)

### 3. 방향 합의
- 온프레미스 로컬 LLM (외부 API 불가)
- A100 + PostgreSQL pgvector로 구축
- 품질팀 15명 대상 파일럿, 생산팀 30명은 2차
- 목표: 복잡 불량 원인 분석 하루 → 2~3시간
