# Phase 3: PLATFORM — 인프라 & 기술 기반

> 고객: 제조기업 (자동차 부품)
> Phase: 3
> 상태: 완료
> 기준: ALFA Framework v1

---

## 3-1. 모델 전략

| 항목 | 결정 | 근거 |
|------|------|------|
| LLM | 로컬 LLM (Ollama) | 온프레미스 필수, 외부 API 불가 |
| 모델 | Llama 3 또는 Mistral (70B급) | A100이면 70B 모델 구동 가능 |
| 한국어 | 한국어 지원 모델 선정 필요 | 불량 이력이 한국어 텍스트 |
| 승격 | 해당 없음 | 이미 로컬. 더 큰 모델 필요 시 GPU 추가 검토 |

## 3-2. 인프라 아키텍처

| 항목 | 결정 |
|------|------|
| 서버 | 기존 A100 서버 활용 |
| LLM 서빙 | Ollama (로컬) |
| DB | 기존 PostgreSQL (품질 DB) |
| 벡터DB | PostgreSQL + pgvector 확장 (동일 인스턴스) |
| GraphRAG | PostgreSQL + Apache AGE (승격 시) |
| 네트워크 | 사내망 전용, 외부 연결 없음 |

> 전부 기존 인프라 위에 소프트웨어만 설치. 추가 하드웨어 구매 없음.

## 3-3. 데이터 파이프라인

| 항목 | 결정 |
|------|------|
| 문서 파싱 | Docling (PDF 검사 기준서 + Word 공정 매뉴얼) |
| 임베딩 | BGE-m3-ko 또는 KURE-v1 (로컬, 한국어 특화) |
| 청킹 | HybridChunker (토크나이저 연동) |
| 불량 이력 | 품질 DB에서 텍스트 필드(증상/원인/조치) 추출 → RAG 인덱싱 |

## 3-4. 보안 인프라

| 항목 | 결정 |
|------|------|
| 가드레일 | 로컬 구현 (입출력 필터링) |
| 데이터 보안 | 온프레미스 → 데이터가 서버 밖으로 안 나감. 자연 충족 |
| 감사 로그 | 로컬 파일 로깅 |

## 3-5. LLMOps

| 항목 | 결정 |
|------|------|
| 모델 관리 | Ollama 모델 버전 관리 |
| 모니터링 | 응답 지연, 검색 정확도, 사용량 (로컬 대시보드) |
| 비용 | 추가 비용 없음 (전기료 수준) |

---

## 핵심: 기존 자산만으로 구성

```
A100 서버
├── Ollama (LLM 서빙)
├── PostgreSQL
│   ├── pgvector (벡터 검색)
│   ├── Apache AGE (GraphRAG, 승격 시)
│   └── 품질 DB (기존, 불량 이력)
└── 로컬 임베딩 모델 (BGE-m3-ko)
```

추가 구매: 없음. 설치만 하면 됨.
