## 기술 메모

- **GraphRAG 강한 시그널:** "버 발생" vs "성형 불량" 동음이의어 + 불량→원인→공정→설비 인과 체인
- **IRM hist 연결:** `docs/IRM/hist/manufacturing-qc/` 시뮬레이션과 직접 연결 가능
- **로컬 LLM:** Ollama + 오픈소스 모델 (Llama, Mistral). A100이면 70B 모델도 가능
- **임베딩:** 로컬 필수 → BGE-m3-ko 또는 KURE-v1
- **벡터DB:** 기존 PostgreSQL에 pgvector + Apache AGE (GraphRAG 시)
- **불량 이력 텍스트:** 표현 비통일 → 엔티티 정규화 이슈 예상

```
Phase 0 WHY:      ✅ 완료 — 불량 대응 개선, 공장장 지시, 3개월
Phase 1 ASSESS:   ✅ 완료 — 온프레미스, A100, PostgreSQL, 속인화 문제
Phase 2 GOVERN:   ✅ 완료 — 외부 반출 금지 원칙만. 업종 규제 가벼움
Phase 3 PLATFORM: ✅ 완료 — A100 + Ollama + PostgreSQL pgvector. 인프라 있음
Phase 4 PILOT:    🔄 진입 가능 — IRM 돌릴 차례
```

> **Phase 0~3이 미팅 1회에서 전부 끝남.** 온프레미스 제약 있지만 A100+PostgreSQL 이미 보유.

## ALFA → IRM 핸드오프 메모

이 케이스는 IRM hist에 `manufacturing-qc` 시뮬레이션이 이미 있다.
유즈케이스 정의서 작성 후 IRM Phase 0을 시작하되, 기존 시뮬레이션 결과를 참조할 수 있다.

```
ALFA 유즈케이스 정의서
  → IRM Phase 0: 리소스 인벤토리 (문서 150개 + 품질DB + MES)
  → IRM Phase 1: 탐색(내부) 중심 + GraphRAG 승격 판단
  → IRM Phase 2: RAG + GraphRAG 구현 체크리스트
```
