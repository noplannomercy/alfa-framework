# 2차 미팅 대화 기록

> 일시: 2026-03-28
> 참석: 김대리 (IT기획팀), 박과장 (컴플라이언스팀), 아키텍트

---

**김대리:** 안녕하세요, 지난번 말씀드린 컴플라이언스팀 박과장님이세요. 아 그리고 문서 목록 메일로 보내드렸는데 확인하셨어요?

**박과장:** 안녕하세요. 김대리한테 대략 들었는데, LLM 도입 관련해서 규제 쪽 확인이 필요하다고요?

**아키텍트:** 네 문서 목록 감사합니다. 박과장님 반갑습니다 — 먼저 전체 그림 간단히 공유드릴게요. 저번에 김대리님과 방향을 잡았는데, 거버넌스 잡고 인프라 세팅하고 내부 직원용 문서 검색으로 파일럿하는 순서입니다. 박과장님 도움이 필요한 건 거버넌스 단계예요.

**박과장:** 아 네, 거버넌스 단계요. 사실 저희도 인공지능기본법 시행되면서 뭔가 준비해야 하는 건 아는데, 구체적으로 뭘 해야 하는지는... 아직 정리가 안 됐어요. 금감원 AI 가이드라인은 작년에 한번 훑어보긴 했는데 그때는 LLM 도입 계획이 없어서 깊게 안 봤고요.

**아키텍트:** 전부 다 할 필요는 없어요. 내부 직원용 문서 검색이 첫 파일럿이라 고객 대면이 아니거든요. 이 범위에서 꼭 확인할 것만 보시죠.

**박과장:** 아 내부 직원용이면 좀 다르죠. 고객한테 뭔가 직접 하는 건 아닌 거죠?

**김대리:** 네, 직원들이 매뉴얼 찾는 걸 도와주는 거예요.

**박과장:** 그 정도면... 그래도 뭐 확인할 건 있을 텐데, 뭐가 있어요?

**아키텍트:** 체크리스트 가져왔어요. 필수 다섯 개, 선택 세 개로 나눴습니다. 필수만 파일럿 전에 확인하면 되고 선택은 나중이에요. 보시겠어요?

```
[필수] — 파일럿 전 완료
  □ AI 의사결정 기구 (누가 승인하는가)
  □ 데이터 분류 (뭘 LLM에 넣어도 되는가)
  □ 접근제어 (누가 쓸 수 있는가)
  □ 인공지능기본법 기본 대응
  □ 금감원 7대 원칙 중 '보조수단성' 확인

[선택] — 확산 전까지
  □ AI 윤리 원칙 상세
  □ 법무 검토 (IP, 책임 소재)
  □ 국제 표준 매핑 (ISO, NIST)
```

내부 직원용 문서 검색이라 필수만 잡으면 시작할 수 있어요. 위에서부터 하나씩 갈까요?

**박과장:** 의사결정 기구... 저희 IT거버넌스 위원회가 있긴 한데, AI 관련 안건을 거기서 다룬 적은 없어요.

**김대리:** 그 위원회에 AI 안건 올리면 되는 거 아닌가요?

**아키텍트:** 기존 위원회 편입 — 좋습니다. 한 가지 더, 이 AI 도입 건에서 누가 최종 승인이고 누가 실행인지 역할도 잡아두면 좋아요. IT기획팀이 실행, 컴플라이언스팀이 검토, 위원회가 승인 — 이 구조 맞아요?

**박과장:** 네, 그 구조가 맞아요. 저희가 검토하고 위원회에서 승인하는 거죠.

**김대리:** 실행은 저희 팀이고요.

**아키텍트:** 좋습니다. 의사결정 구조 확인됐어요. 다음 — 데이터 분류입니다. 이게 제일 중요해요. 박과장님, 사내에 데이터 분류 기준이 있으세요? 어떤 데이터가 외부로 나가면 안 되는지 같은 거요.

**박과장:** 개인정보 등급 분류는 있어요. 개인정보보호법 때문에. 근데 LLM에 뭘 넣어도 되는지 기준은 따로 없어요.

**아키텍트:** 개인정보 등급이 있으면 그거 기반으로 확장하면 돼요. 초안 가져왔는데 — 공개(상품 정보), 내부(매뉴얼, 규정), 민감(고객 개인정보, 신용정보). 민감은 LLM 투입 금지. 기존 개인정보 등급이랑 충돌하는 게 있는지 봐주시겠어요?

**박과장:** 음... 저희 개인정보 등급이 일반/중요/민감 3단계인데, 이거랑 거의 비슷하네요. 공개가 일반, 내부가 중요, 민감이 민감이랑 매핑되겠네요. 충돌은 없어 보여요.

**김대리:** 그러면 저희 200개 문서는 다 내부 등급이니까 넣어도 되는 거죠?

**아키텍트:** 좋아요. 그러면 기존 3단계에 LLM 투입 가능 여부만 추가해서 문서화하면 됩니다. 제가 초안 만들어 드릴게요. 다음 항목 볼까요?

**박과장:** 네, 초안 주시면 검토할게요.

**아키텍트:** 감사합니다. 다음 — 접근제어는 AWS IAM으로 부서별 권한 분리하면 되니까 기술적으로 처리됩니다. 박과장님 쪽에서 별도로 볼 건 없어요.

**박과장:** 네, 그건 IT팀 영역이죠.

**아키텍트:** 그러면 나머지 두 개 빠르게 갈게요. 금감원 7대 원칙 중 '보조수단성' — AI는 보조고 최종 판단은 사람이 한다는 원칙인데, 저희 파일럿이 문서 검색이니까 자연스럽게 충족돼요. 직원이 검색 결과 보고 판단하는 거니까요.

**박과장:** 네, 그건 문제없어요. 고객 상담이나 심사 쪽 가면 그때 깐깐하게 봐야 하지만.

**아키텍트:** 맞습니다. 마지막 — 인공지능기본법. 내부 직원용이라 고영향AI 해당 없고, 투명성은 AI 검색 결과라고 표시하면 되고, 안전성은 가드레일 설정하면 됩니다.

**박과장:** 그 정도면 저도 동의해요.

**김대리:** 그러면 필수 항목 다 된 거예요?

**아키텍트:** 필수 확인 끝났습니다. 로드맵 보여드리기 전에 — 김대리님, 예산이랑 일정 기대가 어떻게 되세요?

**김대리:** 예산은... 윗분이 초기에는 크게 쓰지 말라고 하셨어요. API 비용 수준이면 월 500만원 이내? 인프라 투자는 결과 보고 판단하겠다고요. 일정은... 가능하면 3개월 안에 뭔가 보여달라고 하셨어요.

**아키텍트:** API 비용은 충분한데, 3개월이면 좀 빠듯할 수 있어요. 거버넌스 문서화 2주, 인프라 2~3주, 파일럿 4주면 최소 2개월 반이거든요. 현실적으로 3~4개월 잡으시는 게 안전해요.

**김대리:** 3~4개월... 네 그 정도면 윗분한테 말씀드릴 수 있어요. 3개월 안에 파일럿 돌리고 4개월차에 결과 보고 드리는 거면 되죠.

**박과장:** 저도 그 정도 일정이면 검토 시간 있으니 괜찮아요.

**아키텍트:** 좋습니다. 로드맵 보여드릴게요.

```
1단계 (1~2주): 거버넌스 확정
  - 오늘 확인한 필수 항목 문서화
  - Bedrock 접근 권한 개통

2단계 (2~3주): 인프라 세팅
  - Bedrock + 벡터DB 환경 구성
  - 가드레일 설정 (PII 필터링, 금지 주제)
  - 200개 문서 데이터 파이프라인 구축

3단계 (4~6주): 파일럿
  - 사내 문서 검색 서비스 구축
  - 대상 부서 50명 테스트
  - 2주간 운영 후 평가

4단계 (2~4주): 판정 + 확대 검토
  - 정확도/속도/만족도 평가
  - Go 판정 시 전사 오픈 + 2차 유즈케이스 선정
```

**김대리:** 파일럿 대상 50명은 어디서 뽑아요?

**아키텍트:** IT팀에서 10명, 현업에서 40명 정도로 섞는 게 좋아요. IT팀은 기술 피드백, 현업은 실사용 피드백. 현업은 어느 부서가 제일 적합할까요?

**김대리:** 음... 상담팀이 제일 많이 찾아요. 고객 전화 받으면서 상품 조건 확인하느라 매일 뒤지거든요. 상담팀에서 30~40명 뽑으면 될 것 같아요.

**박과장:** 상담팀이면 좋죠. 거기가 제일 불편해하니까 효과도 바로 보이고.

**아키텍트:** 좋습니다. IT팀 10명 + 상담팀 40명으로 가시죠. 정리합니다. 거버넌스 필수 5개 확인, 예산 월 500만원, 일정 3~4개월, 파일럿 IT 10명 + 상담팀 40명. 김대리님 보고하실 때는 — '거버넌스 확인 완료, 3~4개월 내 파일럿 진행, 비용 월 500만원 이내'로 말씀하시면 됩니다. 이번 주 안에 거버넌스 문서 초안 + 상세 일정 보내드릴게요. 김대리님 Bedrock 신청, 박과장님 문서 검토 부탁드립니다.

**김대리:** 아 보고 멘트까지 감사합니다. 그대로 쓸게요.

**박과장:** 저도 오늘 유익했어요. 문서 오면 바로 검토할게요.

**김대리:** 그러면 바로 시작하는 거죠?

**아키텍트:** 네, 바로 시작합니다. 감사합니다.
