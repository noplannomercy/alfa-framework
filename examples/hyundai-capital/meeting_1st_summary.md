## 논의 내용

### 1. 현재 상황
- 1년 전 RAG PoC 수행 (사내 매뉴얼/규정집 → Qdrant 벡터DB → 검색)
- 검색 정확도 이슈: 동일 용어가 다른 상품 맥락에서 혼재 (예: "한도"가 할부/개인대출에서 다른 의미)
- LLM 기술 급변으로 방향 재설정 필요 → 중단
- 경영진 지시: "LLM 거버넌스 구축해라"

### 2. 인프라/데이터 현황
- AWS 기본 환경 확보 (계정, VPC, 보안 그룹)
- Bedrock 접근 권한 미개통 (신청 예정)
- PoC 때 정리한 PDF 약 200개 잔존 (공유폴더)
  - 대출 상품 매뉴얼, 자동차 할부/리스 안내서, 내부 규정집, 업무 절차서
  - 전부 내부 문서 (고객 정보 없음)

### 3. 사용자/사용 현황
- 사용자: 내부 직원 (상담사, 심사팀, 신입 등)
- 현재 검색 방식: 사내 포털 키워드 검색 → 잘 안 됨 → 공유폴더/동료 문의
- 병목: 문서 찾는 데 시간 과다 소요, 특히 상담사가 고객 통화 중 검색 시

### 4. 방향 합의
- 거버넌스 먼저, 챗봇은 나중
- 내부 직원용부터 시작 (고객 대면은 규제 리스크로 후순위)
- 사내 문서 검색이 첫 파일럿 후보
