**한팀장:** 안녕하세요. 저희 CS팀이 지금 터지고 있어요. 고객 문의가 하루 500건인데 상담사가 15명이라 대응이 안 돼요. 그래서 AI 챗봇을 도입해서 반복 질문은 자동으로 처리하고 싶은데... 어떻게 시작해야 할지 모르겠어요.

**아키텍트:** 챗봇 자체는 가능한데, 고객한테 직접 노출하는 거랑 내부 상담사 보조하는 거랑 접근이 완전 달라요. 어떤 쪽을 생각하고 계세요?

**한팀장:** 음... 처음에는 고객한테 바로 챗봇 붙이는 걸 생각했는데, 그게 좀 위험할 수도 있나요?

**아키텍트:** 위험하다기보다는 준비할 게 달라요. 고객 대면이면 잘못된 답변이 나갔을 때 클레임이 되거든요. 교환 안 된다고 답변했는데 실제로는 가능한 거면? 내부 보조부터 시작하면 상담사가 확인하고 답변하니까 안전해요.

**한팀장:** 아... 그렇구나. 잘못된 답변 나가면 진짜 큰일이긴 하죠. 그러면 상담사 보조부터 하는 게 낫겠네요. 근데 그러면 상담사가 결국 답변하는 거 아니에요? 뭐가 달라지는 거예요?

**아키텍트:** 상담사의 답변 품질이 올라가고 속도가 빨라져요. 구체적으로 보려면 — 지금 상담사가 답변하려면 어디서 정보를 찾아요?

**한팀장:** 상담사들이요... FAQ 페이지가 있긴 한데 오래됐어요. 결국 교환/환불 정책 문서를 공유폴더에서 찾거나, 선임한테 슬랙으로 물어봐요. 상품 정보는 쇼핑몰 어드민에서 보고요. 배송은 택배사 API 조회하고... 다 따로따로예요.

**아키텍트:** 4곳에 흩어져 있으면 그게 병목이죠. 정보가 FAQ, 정책 문서, 어드민, 택배 API — 이 중 제일 시간 잡아먹는 게 뭐예요? 그리고 문서가 대략 몇 개나 돼요?

**한팀장:** 제일 시간 먹는 건 교환/환불 정책이에요. 케이스가 복잡하거든요. "개봉했는데 교환 되나요?", "할인 상품 환불 가능해요?" 이런 거. 정책 문서가 한 30페이지짜리가 있는데 상황마다 달라서 찾기 힘들어요. FAQ는 한 50개, 상품 정보는 어드민에 한 2000개 SKU 있고요.

**아키텍트:** 정책 문서 30페이지 + FAQ 50개 + SKU 2000개. 양은 많지 않아요. 인프라는 어떻게 되세요? 클라우드요?

**한팀장:** AWS 쓰고 있어요. 쇼핑몰 자체가 AWS에서 돌아가거든요. RDS MySQL이랑 EC2요.

**아키텍트:** AWS면 Bedrock 쓸 수 있겠네요. RDS가 MySQL이면 벡터DB는 별도로 필요한데, 그건 설계할 때 잡을게요. 예산이랑 일정 쪽 들어볼게요.

**한팀장:** Bedrock이 뭐예요? 아 뭐 기술적인 건 잘 모르겠고... 예산은 대표님이 월 300만원 정도 쓸 수 있다고 했어요. 빨리 됐으면 좋겠어요. CS팀이 진짜 힘들어서요. 한 달 안에 뭔가 나올 수 있어요?

**아키텍트:** 됩니다. 정리합니다 — FAQ 50개 + 정책 문서 30페이지 + SKU 2000개, AWS 인프라 있고, 월 300만원. 상담사 보조로 먼저 한 달, 안정화 후 고객 대면 전환 검토. 상담사 15명 전원 파일럿. 이번 주 안에 상세 일정 보내드릴게요. 한팀장님은 정책 문서랑 FAQ 파일만 공유해 주세요.

**한팀장:** 오 깔끔하네요. 근데 나중에 고객 대면으로 전환할 때 뭘 더 확인해야 하는 거예요?

**아키텍트:** 고객 대면이면 두 가지요. 첫째 — 오답 나갔을 때 책임. '교환 안 된다'고 챗봇이 답했는데 실제로 되는 거면 클레임이니까요. 둘째 — 개인정보. 고객이 주문번호 넣고 질문하면 개인정보 처리가 되거든요. 이건 상담사 보조가 잘 돌아간 다음에 같이 정리하면 됩니다. 지금 신경 안 쓰셔도 돼요.

**한팀장:** 아 그렇구나. 알겠어요. 그러면 바로 시작하죠!

**아키텍트:** 좋습니다. 한팀장님은 정책 문서랑 FAQ 공유만 부탁드리고, 상세 일정 이번 주에 보내드릴게요.
