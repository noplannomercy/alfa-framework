## 기술 메모

- **RDS MySQL → 벡터DB 별도 필요.** pgvector는 PostgreSQL 전용. 옵션:
  - OpenSearch Serverless (AWS 네이티브)
  - RDS PostgreSQL 새로 만들어서 pgvector
  - Bedrock Knowledge Base (관리형)
- FAQ 50개 + 정책 30페이지 → 데이터 양 적음. 기본 RAG로 충분
- **GraphRAG 필요성: 낮음** — 교환/환불 정책은 조건 분기지 인과 관계가 아님
- **고객 대면 전환 시 새 변수:**
  - 오답 책임 (법무 검토 필요)
  - 개인정보 처리 (주문번호 → 고객 정보 연결)
  - 이때 ALFA Phase 2 GOVERN [선택]→[필수] 격상
