## 기술 메모 (내부)

- MySQL → 벡터DB 별도 필요 (OpenSearch Serverless 또는 RDS PostgreSQL+pgvector 신규)
- 데이터 소량 (FAQ 50 + 정책 30p + SKU 2000) → 기본 RAG 충분
- GraphRAG 불필요 — 교환/환불은 조건 분기, 인과 관계 아님
- **고객 대면 전환이 이 케이스의 핵심 변수** — 2차에서 ALFA Phase 2 재방문
