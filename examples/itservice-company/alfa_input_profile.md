## [2] 현재 AI/LLM 상태

- **AI/LLM 도입 경험:** 없음 (제로)

- **기존 도구:** Confluence (문서), Slack (커뮤니케이션), GitHub (코드)

## [3] 인프라 현황

- **클라우드:** AWS (프로덕션 서비스 운영 중)
- **기존 환경:** EKS, RDS PostgreSQL
- **LLM 서비스:** Bedrock (개통 신청 예정)
- **벡터DB:** RDS PostgreSQL + pgvector 확장 (추가 비용 없음)
- **네트워크:** 일반 (금융/폐쇄망 수준 제약 없음)
- **보안 등급:** 일반

---

## [4] 규제 환경

- **적용 규제:** 특별한 업종 규제 없음
- **보안 제약:** 고객사 SI 프로젝트 문서 중 고객사 정보 포함분 제외 (NDA)
- **데이터 분류:** 2분류
  - 사내 문서: Confluence 기술 문서, 운영 매뉴얼, 온보딩, 규정 → LLM 투입 가능
  - 고객사 정보: 고객사 소스코드, 인프라 정보, NDA 문서 → **LLM 투입 금지**
- **컴플라이언스 별도 검토:** 불필요

---

## [5] 하고 싶은 것

- **스폰서:** CTO (구두 승인 — "해봐, 비용 많이 쓰지 마")
- **현업 니즈:** 사내 지식 검색
  - 현재: Confluence 키워드 검색 → 원하는 문서 못 찾음 → 슬랙에서 동료한테 질문
  - 병목: 같은 질문 반복, 시니어에게 질문 집중
- **1차 파일럿:** Confluence 사내 기술 문서 의미 기반 검색
- **2차 확장:** 슬랙 #dev-help 채널 데이터 추가
- **사용자:** 전 직원 (개발자 중심, 300명)

---

## [6] 제약 사항

- **예산:** 월 200만원 이내 (API 비용)
- **일정:** 한 달 (2주 프로토타입 + 2주 피드백)
- **내부 인력:** 이과장 (개발팀 리드) + 개발자 조직이라 자체 기술력 보유
- **외부 파트너:** 아키텍트 (설계+초기 구축)

## 기술 메모 (내부)

- GraphRAG 필요성: 낮음 — 개발 문서는 용어 충돌 적고 질의가 구체적
- 슬랙 데이터: 노이즈 필터링 필요 (잡담/이모지/스레드 정리)

- "배포 에러" 히트 수십 개 문제 → 메타데이터 필터링 (서비스명, 도구명) 또는 리랭킹
