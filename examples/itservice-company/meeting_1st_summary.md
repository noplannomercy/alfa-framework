## 논의 내용

### 1. 현재 상황
- Confluence에 사내 기술 문서 500페이지+ (API 문서, 아키텍처, 트러블슈팅, 운영 매뉴얼, 온보딩, 규정)
- Confluence 키워드 검색 한계 → 슬랙에서 동료한테 물어보는 게 더 빠름
- 슬랙 #dev-help 채널에 트러블슈팅 답변 축적 (비공식 지식 창고)
- CTO 승인: "해봐, 비용 많이 쓰지 마"

### 2. 인프라/데이터
- AWS 프로덕션 환경 운영 중 (EKS, RDS PostgreSQL)
- Bedrock 미사용 (개통 신청 예정)
- RDS PostgreSQL → pgvector 확장으로 벡터DB 추가 비용 없음
- 문서: Confluence 500페이지+ (사내 문서, 고객 정보 없음)
- 슬랙: #dev-help 채널 (2차 데이터 소스)

### 3. 규제/보안
- IT서비스 — 금융/의료 수준 규제 없음
- SI 프로젝트 문서 중 고객사 정보 포함분 제외
- 사내용으로만 사용 시 NDA 이슈 없음

### 4. 방향 합의
- 1차: Confluence 문서 기반 사내 지식 검색
- 2차: 슬랙 #dev-help 데이터 추가
- 고객사 정보 포함 문서 제외
