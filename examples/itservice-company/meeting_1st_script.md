# 1차 미팅 대화 기록

> 일시: 2026-03-25
> 참석: 이과장 (개발팀 리드, DevOps 겸임), 아키텍트

---

**이과장:** 안녕하세요. 저희가 요즘 LLM 도입을 검토하고 있는데요. 사내 지식 공유가 문제예요. Confluence에 문서 쌓아놔도 아무도 안 찾아보고, 결국 슬랙에서 물어보거든요. 이걸 LLM으로 풀 수 있을 것 같아서요.

**아키텍트:** Confluence에 어떤 종류의 문서가 있어요? 개발 문서, 운영 매뉴얼, 고객 가이드... 이런 게 섞여있어요?

**이과장:** Confluence에 개발 문서가 제일 많아요. API 문서, 아키텍처 설계서, 트러블슈팅 가이드... 그리고 운영 매뉴얼, 온보딩 가이드, 사내 규정도 있고요. 한 500페이지 넘을 거예요. 근데 검색이 안 돼서 다들 슬랙에서 "이거 어디있어?" 하고 물어봐요.

**아키텍트:** Confluence 자체 검색으로 안 되는 거예요? 아니면 검색은 되는데 원하는 게 안 나오는 거예요?

**이과장:** Confluence 검색 자체가 키워드 매칭이라... "배포 에러"로 검색하면 배포 관련 문서 수십 개가 다 나와요. 근데 내가 원하는 건 "Jenkins 배포 시 타임아웃 에러 해결법"이거든요. 결국 슬랙에서 시니어한테 물어보는 게 더 빨라요.

**아키텍트:** 슬랙에서 물어보는 게 더 빠르다는 건, 슬랙에 답변이 쌓여있다는 거잖아요. 슬랙 이력도 검색 대상에 넣을 생각 있어요?

**이과장:** 오 슬랙이요? 생각 못 했는데... 사실 슬랙 #dev-help 채널에 트러블슈팅 답변이 엄청 쌓여있어요. 거기가 진짜 지식 창고인데 검색이 안 되니까 똑같은 질문이 반복돼요. 그것도 넣을 수 있어요?

**아키텍트:** 좋은 포인트예요. 슬랙 데이터는 나중에 같이 설계할 때 다루고요 — 먼저 몇 가지 더 확인할게요. 이걸 위에서 지시한 건가요, 팀 차원에서요?

**이과장:** 팀 차원이에요. CTO님한테 "LLM으로 사내 지식 검색 한번 해보자" 제안했더니 "해봐, 근데 비용 많이 쓰지 마" 이러셨어요. ㅋㅋ 그래서 일단 알아보는 중이에요.

**아키텍트:** 알겠어요. IT서비스라 금융이나 의료처럼 규제가 빡센 건 아닐 텐데 — 고객사 데이터를 다루는 게 있어요? NDA나 보안 제약이요.

**이과장:** 아... 고객사 소스코드나 인프라 정보는 좀 민감하죠. SI 프로젝트 문서 중에 고객사 정보가 포함된 게 있어요. 그건 빼야 할 것 같고요. 사내 기술 문서, 자체 솔루션 문서, 운영 매뉴얼 이런 건 괜찮아요. NDA는 프로젝트별로 걸려있는데, 사내용으로만 쓰면 문제없을 거예요.

**아키텍트:** 깔끔해요. 사내 문서만 대상, 고객사 정보 제외. 금융이나 의료가 아니니까 거버넌스가 가벼워요. 그러면 인프라랑 예산만 확인하면 오늘 정리될 것 같아요.

**이과장:** 네, 저희가 규제 환경은 편한 편이죠. AWS는 이미 프로덕션 서비스 돌리고 있어서 환경은 다 있어요. EKS도 쓰고 있고, RDS PostgreSQL도 있고요. Bedrock은 안 써봤는데 열면 되죠. 예산은... CTO가 월 200만원 이내로 해보라고 했어요. 그 안에 되나요?

**아키텍트:** 여유 있어요. 게다가 RDS PostgreSQL 이미 있으시면 pgvector 확장만 켜면 벡터DB가 공짜예요. Bedrock API만 비용이고 월 200이면 넉넉합니다. 일정은 어떻게 보고 계세요?

**이과장:** 빨리 되면 좋죠. 한 달 안에 프로토타입 보여줄 수 있어요? 개발자들이니까 바로 피드백 줄 수 있거든요.

**아키텍트:** 개발자 조직이니까 빠르게 갈 수 있어요. 2주 안에 Confluence 문서로 프로토타입 띄우고, 나머지 2주에 피드백 받으면서 개선하면 한 달에 파일럿까지 가능해요. 정리합니다. Confluence 500페이지 사내 기술 문서, 고객사 정보 제외, AWS 인프라 다 있고 RDS PostgreSQL에 pgvector, Bedrock API 월 200만원. 개발자 조직이라 거버넌스도 가볍고요. 2주 프로토타입, 2주 피드백으로 한 달에 파일럿까지. 이번 주 안에 상세 일정 보내드릴게요. 이과장님은 Bedrock 권한 신청이랑, Confluence에서 고객사 정보 포함된 문서 목록만 추려주세요 — 그거 빼고 나머지 넣으면 됩니다.

**이과장:** 깔끔하네요. 슬랙 데이터는요?

**아키텍트:** 슬랙은 2차로요. Confluence 먼저 돌려보고 효과 확인한 다음에 슬랙 #dev-help 채널 데이터 추가하면 됩니다.

**이과장:** 오케이. 바로 갑시다.
