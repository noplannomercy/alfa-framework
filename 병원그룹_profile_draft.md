# 병원그룹 고객 프로필 (확정)

> 작성일: 2026-03-25
> 상태: **3차 미팅 완료 — 6항목 모두 🟢, PoC 확정**

## 1. 회사 개요
- 업종: 의료 (병원 그룹)
- 규모: 중견 — 서울 본원 + 지방 분원 3개
- 주력 사업: 종합병원 운영
- 의사결정 기구: **의료정보위원회** — AI 관련 안건 심의 [2차 미팅 확인]
- RACI: 실행=의료정보팀, 검토=영상의학과, 승인=의료정보위원회, 보고=CTO [2차 미팅 확인]

## 2. AI/LLM 현황
- 기존 시도: **작년 외부 업체 챗봇 PoC — 중단** [3차 미팅 확인]
- 실패 원인: 범용 LLM이 의학용어를 이해하지 못해 정확도 낮음 [3차 미팅 확인]
  - "의학용어를 못 알아들어서 엉뚱한 답변이 나왔어요"
- CTO 방향: **"이번엔 우리 데이터로 직접, 외부 업체 의존 줄이고 싶다"** [3차 미팅 확인]
- GPU 인프라: A100 x2 (연구팀 소유, 공유 합의 완료)

## 3. 인프라
- 클라우드/온프레미스: **온프레미스 확정** — 데이터 외부 반출 절대 불가, 외부 API 불가
- EMR: **자체개발**, 온프레미스 서버 운영
  - 분원 3곳은 EMR 버전이 다름, 본원과 데이터 구조 차이
  - 접근권한: 기존 EMR 권한체계 따름
  - 감사로그: 기존 EMR 감사로그에 AI 조회 이력 추가
- PACS: **Infinitt 제품**, HL7 FHIR 연동 가능 확인 [3차 미팅 확인]
  - 연동: 읽기전용만 허용
- GPU: NVIDIA A100 x2 — **주중 업무시간 우선 사용 가능** (야간/주말은 연구팀) [3차 미팅 확인]
- 데이터 보관: 원내 서버, 학습용 데이터셋은 별도 격리 스토리지

## 4. 규제 환경
- 감독기관: 식약처(MFDS), 보건복지부, 개인정보보호위원회
- 컴플라이언스: 의료정보보안팀장이 규제 검토 담당
- 데이터 분류: 개인건강정보(PHI) — 가명처리 후 LLM 투입 가능 (내부 검토 완료)
- 가명처리 기준: 보건복지부 가이드라인 — 이름/생년월일/주민번호 마스킹
- IRB: **간편심의 확정, 2주 통과 예상** (후향적 데이터)
- 식약처 인허가: **보조수단 인정** — "AI 결과를 의사가 최종 확인하는 구조"
- 환자 대면: 현재 없음, 향후 확장 의향 (2단계)
- 외부 API: 절대 불가

## 5. 원하는 것
- 경영진 지시: CTO "AI 도입 검토" → **"우리 데이터로 직접 해보자"**
- 현업 니즈:
  - **1순위 (PoC 확정)**: 본원 영상의학과 **판독문 검색 보조** [3차 미팅 최종 합의]
  - **2순위**: 간호기록 요약 — 교대 인수인계용, 현재 수기 30분
- 핵심 Pain Point:
  - 의사별 표현 차이로 판독문 검색 불가 (CT/컴퓨터단층촬영/영문약어 혼재)
  - 간호 인수인계 수기 30분
- 장기 비전: 환자 설명용 활용

## 6. 제약사항
- 예산: **PoC 5천만원 이내**
- 일정: **4월 중순 PoC 시작 목표** (IRB 2주 + 데이터 준비 2주 감안) [3차 미팅 확인]
- 인력: **의료정보팀 3명** 투입
- 범위: **본원만**, 분원 데이터 1차에서 제외
- 고객 대면: 의사/간호사 전용
- 의사결정: CTO 스폰서, 의료정보위원회 승인

## 내부 메모

### 거버넌스
- 무게: **Heavy (관리 가능, 경로 완전 확보)**
  - 가명처리 기준 확정 (보건복지부 가이드라인)
  - IRB 간편심의 2주
  - 보조수단 논리 확보 (의료정보보안팀장)
  - 의료정보위원회 심의 체계
  - **PoC 진행을 막는 규제 블로커 없음**

### 감지 신호 (최종)
- **#GraphRAG (강)**: 의사별 표현 차이 + 용어 표기 불일치 + 이전 PoC 실패 원인과 동일
- **#on-premise (확정)**: 데이터 반출 불가 + 외부 API 불가
- **#poc-history**: 작년 범용 LLM 챗봇 PoC 실패 → 의학용어 미이해가 원인
  - 이번 PoC 차별점: 자체 데이터 기반 + 용어 온톨로지/GraphRAG
- **#2-phase**: 환자 설명용 확장 의향 (현재 1단계)
- **#gov-cleared**: 모든 규제 경로 확보

### PoC 설계 (확정)
- **범위**: 본원 영상의학과 판독문 검색 보조
- **인프라**: A100 x2 (주중 업무시간), 온프레미스, 로컬 LLM
- **PACS**: Infinitt, HL7 FHIR 읽기전용 연동
- **데이터**: 후향적 판독문 → 가명처리 → LLM 투입
- **권한**: EMR 권한체계 연동
- **감사**: EMR 감사로그 + AI 조회 이력
- **학습 데이터**: 격리 스토리지
- **예산**: 5천만원 이내
- **인력**: 의료정보팀 3명 + 외부
- **일정**: 4월 중순 시작
- **핵심 차별화**: GraphRAG/용어 온톨로지 → 이전 PoC 실패 원인 직접 해결

### 예상 로드맵
- IRB 심의: ~2주 (3월 말 ~ 4월 초)
- 데이터 준비: ~2주 (가명처리 + 격리 스토리지 세팅)
- **PoC 시작: 4월 중순**
- Phase 1 (PoC): 본원 판독문 검색 보조 → 2~3개월
- Phase 1.5: 간호기록 요약 데모 → +2주
- Phase 2: 분원 확대 (별도 예산)
- Phase 3: 환자 대면 (거버넌스 재검토)
