# 유즈케이스 정의서: 판독문 검색 보조

> 고객: 병원그룹
> 작성일: 2026-03-25
> 상태: 확정 (3차 미팅 합의)

## 개요
- 목적: 영상의학과 판독문을 의학용어 동의어/유사표현을 반영하여 정확하게 검색
- 사용자: 영상의학과 전문의, 임상의
- 입력: 자연어 검색 쿼리 (예: "폐 결절 소견", "lung nodule", "CT 소견")
- 출력: 관련 판독문 목록 (유사 표현 포함 매칭), 핵심 소견 하이라이트

## 행위 유형 분포 (예상)

| 행위 유형 | 비중 | 설명 |
|----------|------|------|
| 탐색 | **60%** | 유사 소견/사례 검색, 의학용어 동의어 확장 검색 |
| 접근 | **25%** | 특정 환자/기간/모달리티 판독문 직접 조회 |
| 실행(유연) | **10%** | 검색 결과 기반 소견 비교, 참고 자료 정리 |
| 실행(확정) | **5%** | 특정 포맷 리포트 출력 |

## 리소스 예상

| 리소스 유형 | 예시 | 규모 |
|------------|------|------|
| 문서/파일 | 판독문 (텍스트) | 수만~수십만 건 (본원 후향적 데이터) |
| 시스템/DB | EMR (자체개발), PACS (Infinitt) | 2개 시스템 연동 |
| 용어 체계 | 의학용어 동의어 사전, SNOMED-CT, KCD | 온톨로지 구축 또는 기존 표준 활용 |

## 데이터 파이프라인

```
PACS (Infinitt)          EMR (자체개발)
    │ HL7 FHIR                │
    │ (읽기전용)               │
    ▼                         ▼
  영상 메타데이터          판독문 텍스트
    │                         │
    └─────────┬───────────────┘
              ▼
      가명처리 (이름/생년월일/주민번호 마스킹)
              │
              ▼
      격리 스토리지 (학습용)
              │
              ▼
      임베딩 + 용어 온톨로지 매핑
              │
              ▼
      벡터DB (로컬)
              │
              ▼
      RAG 파이프라인 + 로컬 LLM (A100 x2)
              │
              ▼
      검색 결과 (EMR 권한체계 연동, 감사로그 기록)
```

## GraphRAG 필요성
- **필요 (강)**
- 근거:
  - "같은 소견인데 의사마다 표현이 달라서 검색이 안 됨"
  - "CT, 컴퓨터단층촬영, 영문약어 등 표기가 제각각"
  - 이전 PoC 실패 원인이 바로 이 문제 (범용 LLM의 의학용어 미이해)
- 접근 방식:
  - 의학용어 온톨로지 (SNOMED-CT, KCD 등 표준 활용)
  - 동의어/유사표현 그래프 구축
  - 쿼리 확장: 사용자 입력 → 동의어 확장 → 벡터 검색

## 성공 기준 (PoC)
- 판독문 검색 시 동의어/유사표현 포함 정확도 80% 이상
- 기존 EMR 키워드 검색 대비 재현율(recall) 유의미한 개선
- 영상의학과 전문의 만족도 조사 통과
- 검색 응답 시간 5초 이내

## 참고: 이전 PoC 실패 교훈
- 범용 LLM + 일반 프롬프트 → 의학용어 불일치로 정확도 미달
- 이번 차별점: 자체 판독문 데이터 기반 임베딩 + 용어 온톨로지 + GraphRAG
